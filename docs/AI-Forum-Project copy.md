# AI 经验分享社区 — 项目基本设想

## 一、产品概述

**一句话定位**：面向南京大学学生的 AI 使用经验分享平台，让学生通过真实案例学习如何高效使用 AI。

**核心理念**：当前 AI 工具的使用经验分享主要来自自媒体和资深程序员，大学生的日常提效经验（写论文、做实验报告、写代码作业等）缺少分享渠道。本平台聚焦"校内场景"，用真实的学生经验帮助更多同学学会用 AI。

**产品名称**：待定（建议取一个有校园感、年轻化的名字）

## 二、目标用户

- **主要用户**：南京大学在校学生
- **用户画像**：有一定技术基础但不精通 AI 工具的大学生，希望通过他人经验快速上手
- **内容生产者**：对 AI 使用有心得的学生，愿意分享自己的 Prompt 和使用技巧

## 三、核心交互循环

```
浏览帖子 → 发现好用的 Prompt → 一键复制试用 → 回来标记"已复现"
```

## 四、功能模块

### MVP（中期检查前必须完成）

#### 1. 用户模块

- 登录（模拟登录或邮箱登录，不需要真实第三方审核）
- 基础个人资料：昵称、头像、院系、简介
- **声望系统**：
  - 声望值来源：发帖获已复现（+10/次）、被点赞（+2/次）、签到（+1/天）
  - 等级称号：AI 新手（0-49）→ Prompt 猎人（50-199）→ 提效大师（200-499）→ AI 巫师（500+）
  - 等级头衔显示在帖子和评论旁边，给内容生产者正反馈
  - 个人中心展示声望变化趋势

#### 2. 帖子/内容模块

- Markdown 编辑器，支持代码高亮、图片上传
- 帖子分类：学术提效 / AI 绘图 / 代码编写 / 其他
- 话题标签（Tag）
- 帖子详情页：正文渲染 + 文章目录自动生成

#### 3. Prompt 卡片（核心亮点）

- 帖子中可插入"Prompt 卡片"组件
- 卡片字段：适用模型、Prompt 正文、系统设定、参数说明
- **一键复制**功能
- 卡片视觉设计要突出，区别于普通正文

#### 4. 互动模块

- 评论（楼中楼回复）
- 点赞、收藏
- **"已复现"按钮**：标记有多少人成功使用了该方案

#### 5. 首页与浏览

- 首页推荐流（最新 / 最热 / 已复现最多）
- 分类筛选
- 搜索

#### 6. 通知系统（轻量版）

- 触发场景：被点赞、被标记"已复现"、被回复/被 @、声望升级
- 数据库表：`notifications`（接收者、类型、关联对象、是否已读）
- 进站轮询未读数（无需 WebSocket）
- 导航栏右上角红点 + 未读数
- 消息中心页：分类列表 + 已读/未读切换

### 后续迭代（中期检查之后）

- 悬赏问答模块（积分机制）
- Prompt 变量占位符（用户可在线修改 Prompt 参数后复制）
- Fork 与版本控制（复制他人 Prompt 二次修改）
- 即时私信
- 校内邮箱验证（强化南大身份认证）
- 后台管理与内容审核

## 五、技术方案

| 层 | 选型 | 理由 |
|---|------|------|
| 前端框架 | Next.js (App Router) | 全栈一体，SSR 支持，开发速度快 |
| UI 样式 | TailwindCSS + shadcn/ui | 组件质量高，适合快速搭建 |
| 数据库 | Supabase (PostgreSQL) + Prisma | 云数据库，免费套餐够用，Vercel 官方推荐搭配 |
| 部署 | Vercel | Next.js 原生支持，一键部署 |
| Markdown 编辑器 | MDX Editor 或 Tiptap | 支持自定义组件嵌入（Prompt 卡片） |
| Markdown 渲染 | react-markdown + rehype 插件 | 代码高亮、目录生成 |

## 八、中期检查演示要点

评委关注的三个问题，demo 要直接回答：

1. **这是什么？** — 30 秒讲清楚产品定位
2. **创意在哪？** — 重点展示 Prompt 卡片 + "已复现"机制 + 声望等级体系
3. **能不能用？** — 现场演示：浏览帖子 → 打开 Prompt 卡片 → 一键复制 → 标记已复现 → 查看声望变化
